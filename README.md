# Портфолио (статический фронтенд)

Персональный сайт: данные из `me.txt`, `stack.txt`, `projects.txt` в корне репозитория `test`. Стиль — **нео-брутализм** (жирные границы, смещённые тени, контраст, отсылка к конструктивистской графике).

## Установка

```bash
cd portfolio
npm install
```

## Запуск локально

```bash
npm run dev
```

Откройте URL из терминала (обычно `http://localhost:5173`).

## Сборка

```bash
npm run build
```

Артефакты — в каталоге `dist/`.

Просмотр прод-сборки:

```bash
npm run preview
```

## Деплой

### Vercel

1. Подключите репозиторий к [Vercel](https://vercel.com/).
2. **Root Directory** укажите `portfolio` (если монорепозиторий с корнем `test`).
3. **Build Command:** `npm run build`
4. **Output Directory:** `dist`
5. **Install Command:** `npm install`

Либо из каталога `portfolio`:

```bash
npx vercel
```

### Netlify

- Build: `npm run build`
- Publish directory: `dist`

### GitHub Pages

Структуру проекта менять не нужно: Vite собирает статику в `portfolio/dist/`.

- **Project Pages** (`https://<user>.github.io/<repo>/`): при сборке задаётся `base: /<repo>/`. В репозитории с корнем `test` уже есть workflow [`.github/workflows/deploy-pages.yml`](../.github/workflows/deploy-pages.yml): он собирает приложение из `portfolio/`, выставляет `GITHUB_PAGES_BASE` и заливает артефакт в Pages. В настройках репозитория включите **Settings → Pages → Source: GitHub Actions**.
- **Сайт пользователя** (репозиторий `<username>.github.io`): workflow сам подставит `base: /`.
- Ручная сборка: `GITHUB_PAGES_BASE=/<имя-репозитория>/ npm run build`, затем выложите содержимое `dist/` (ветка `gh-pages`, или «Deploy from branch» с папкой `dist` — как настроите).
- Если корень репозитория — сам каталог `portfolio` (без вложенной папки), поправьте workflow: уберите `working-directory: portfolio`, путь артефакта задайте `dist`, `cache-dependency-path` — `package-lock.json`.
- **Уже есть репозиторий с заглушкой:** да, можно перенести **содержимое** папки `portfolio` в корень того репо (не саму папку `portfolio`). Удалите или замените старый `index.html` и лишние файлы заглушки. Дальше — `npm install`, `npm run build`, деплой из `dist/`; workflow настройте под корень, как в пункте выше.

Подробнее: [Vite: GitHub Pages](https://vite.dev/guide/static-deploy.html#github-pages).

## Резюме (PDF)

Положите файл `resume.pdf` в `portfolio/public/`, чтобы ссылка «Скачать PDF» в блоке контактов работала. Имя в репозитории может отличаться — скопируйте файл как `resume.pdf` или обновите путь в `src/data/content.ts` (`resumePath`).

## Дизайн и стек

### Выбранный стиль: нео-брутализм

Среди вариантов (биоморфизм, webcore, неоклассика, шинуазри, брутализм) для веб-портфолио больше всего готовых материалов у **брутализма / нео-брутализма**: CSS-библиотеки в духе Neubrutalism / NeoBrutalismCSS, десятки шаблонов портфолио и лендингов на GitHub, паттерны на Tailwind и чистом CSS. У **webcore** сильная ниша (Astro + Webcore UI), но экосистема уже привязана к одному UI-kit. **Неоклассика** и **шинуазри** на GitHub почти не представлены как переиспользуемые UI-системы для портфолио. **Биоморфизм** чаще смешивают с нейморфизмом и даёт меньше прямых референсов именно для лендингов.

### Технологии

**Vite + React + TypeScript** — быстрая сборка, статический вывод без бэкенда, предсказуемый деплой на Vercel/Netlify, компонентная структура без лишних зависимостей.

### Шрифты

- **Archivo Black** — плотные заголовки в духе афиш и плакатов, характерных для грубого / нео-бруталистского веба.
- **IBM Plex Mono** — моноширинный текст: ассоциация с кодом и терминалом, хорошая читаемость для описаний и навигации.

Оба подключаются через Google Fonts с `display=swap`.

### Идея макета

Сетка на фоне, карточки с «жёсткой» обводкой и смещённой тенью, акцентный красный и лаймовый блоки для ритма, бегущая строка с ключевыми словами стека, таймлайн образования, сетка тегов для технологий и карточки проектов с явной ссылкой на GitHub.
